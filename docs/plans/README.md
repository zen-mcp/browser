# Browser MCP Improvement Plans

This directory contains phase-based implementation plans for evolving `zen-mcp/browser` from a compact MVP into a production-grade, personal Browser MCP server with first-class authentication, policy controls, session profiles, observability, and quality gates.

## Phase sequence

1. [Phase 01 — Auth, Policy, and Configuration](./phase-01-auth-policy/plan.md)
2. [Phase 02 — AI-Friendly Browser Snapshot and Element References](./phase-02-snapshot-refs/plan.md)
3. [Phase 03 — Personal Session Profiles and Website Auth Workflows](./phase-03-session-profiles/plan.md)
4. [Phase 04 — Debug Artifacts, Network Visibility, and Run Bundles](./phase-04-debug-artifacts/plan.md)
5. [Phase 05 — Quality Gates, Integration Tests, and Release Hardening](./phase-05-quality-release/plan.md)

Each `plan.md` is written as an implementation handoff: goal, architecture, acceptance criteria, file paths, tasks, tests, and verification commands.
